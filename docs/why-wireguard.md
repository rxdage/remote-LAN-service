# 为什么要 WireGuard?(RustDesk 自己有服务器,为何还加 WG)

短答:**A 档能用 RustDesk,但 WG 给你的是 RustDesk 给不了的三样东西。**

## 两种连接方式

**A 档 —— 自建 RustDesk 服务器,按 ID 连**
- 服务器跑 RustDesk 专用信令(hbbs)+ 中继(hbbr);
- 两端按 ID 注册,连接时服务器居中撮合:先试公网 UDP 打洞直连,打不通退到中继;
- 能用。但:① 发起连接**依赖 hbbs 活着**;② 就**一条路**;③ 只能跑 RustDesk。

**B 档 —— WireGuard 直连内网 IP**
- 服务器跑通用 WireGuard,每台机器进**同一个虚拟局域网**、有个内网 IP(如 10.66.0.2);
- RustDesk 填对方内网 IP 直连,**完全不碰任何 RustDesk 服务器**。

## WG 多给你的三样

| | A(按 ID) | B(WG 直连) |
|---|---|---|
| 发起连接依赖 | 要 hbbs 撮合 | **只要隧道通,无需信令服务器** |
| 能跑什么 | 只 RustDesk | **RDP / SMB / SSH / 任何服务**都能跑(它是通用内网) |
| 有几条路 | 一条 | **可多条独立隧道**(再加 ZeroTier),坏一条换个 IP 继续 |
| 加密边界 | RustDesk 自己的 | WG 网络层加密(标准、你的密钥;服务器中继时也看不到内容) |
| 搭建成本 | 极简 | 每台要配 WG 密钥(更强但更麻烦) |

## 怎么选
- **只想自建 RustDesk、日常远程** → A 够了,别折腾。
- **要抗断**(一条线挂了还能进)、**要在这些机器间跑别的服务**(RDP/文件共享)、**要多路冗余** → 上 B。
- **连服务器整机挂掉都要扛** → C(第二台异地服务器)。

三档是叠加的:A 起步,随时加模块升到 B/C,不用重来。
